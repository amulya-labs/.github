# Contributing to Amulya Labs

Thanks for your interest in contributing! Here's a quick guide:

## Process

1. **Fork** the repository
2. **Create a branch** for your changes (`git checkout -b feature/my-feature`)
3. **Make your changes** (keep PRs focused and small)
4. **Push** to your fork
5. **Open a Pull Request** and reference any related issues

## Guidelines

- **Small, focused PRs** are easier to review and merge
- **Reference issues** in your PR description (e.g., "Closes #123")
- **Follow the existing code style** in the project
- **Add tests** if applicable
- **Update documentation** if behavior changes

## Questions?

Open an issue for bugs, feature requests, or questions. We're here to help!
